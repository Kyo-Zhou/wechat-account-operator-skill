# Layout System

Use this reference when building or revising a WeChat article layout.

## Target Feel

For insurance long-form content, aim for:

- editorial, not promotional
- calm, not noisy
- structured, not rigid
- premium, but still native to WeChat

## Recommended Layout Stack

### First Screen

Include only:

- column label
- one strong opening statement
- one supporting sentence
- one compact conclusion or reading guide block

Avoid oversized hero-card styling unless the topic is highly emotional.

### Section Headers

Best default for this domain:

- calm color palette
- clear hierarchy
- one recognizable visual marker

Good options:

- editorial header with thin divider
- compact colored label block
- subtle left marker plus short heading

Avoid heavy banner bars on every section.

### Callout Modules

Use sparingly:

- `先看结论`
- `风险提示`
- `关键判断`
- `给读者的行动建议`

If every paragraph becomes a card, the article starts to feel templated.

### Paragraph Rhythm

Prefer:

- medium paragraph length
- visible spacing between paragraphs
- occasional quoted takeaway

Avoid:

- dense walls of text
- over-styled inline highlighting
- too many centered slogans

## Reference-Inspired Rules

From good public WeChat layouts, the most reusable patterns are:

- a stronger first screen than the rest of the article
- section headers that anchor scanning
- only a few high-quality highlight blocks
- restrained color use
- clean closing section

## What To Avoid

- debug-preview style wrappers
- generic Markdown-export look
- too many gradients
- too many badges
- overuse of quotation marks or colored inline phrases
